# 기술 선택 이유 (Flask vs Django)

## 왜 Flask를 선택했나요?

### 1. 프로젝트 요구사항에 적합
- **REST API만 필요**: 이 프로젝트는 주로 REST API 엔드포인트만 필요합니다
- Django는 풀스택 웹 프레임워크로 템플릿, 관리자 페이지 등 많은 기능을 포함하지만, 이 프로젝트에서는 불필요합니다

### 2. 가벼움과 빠른 개발
- Flask는 **마이크로 프레임워크**로 최소한의 기능만 제공
- 필요한 것만 추가하는 방식 (필요한 패키지만 설치)
- 빠른 프로토타이핑에 적합

### 3. 단순한 구조
- 파일 구조가 간단하고 이해하기 쉬움
- 작은 프로젝트에 적합한 구조

### 4. 유연성
- 원하는 방식으로 구조화 가능
- ORM을 사용하지 않아도 됨 (이 프로젝트는 간단한 데이터 처리)

## Flask vs Django 비교

| 항목 | Flask | Django |
|------|-------|--------|
| **용도** | 마이크로 프레임워크, API | 풀스택 웹 프레임워크 |
| **학습 곡선** | 낮음 | 높음 |
| **설정** | 최소한 | 많은 설정 필요 |
| **ORM** | 선택사항 (SQLAlchemy) | 내장 (Django ORM) |
| **관리자 페이지** | 없음 (별도 구현) | 내장 |
| **템플릿 엔진** | Jinja2 (선택사항) | 내장 |
| **유연성** | 높음 | 낮음 (규칙이 많음) |
| **용량** | 작음 | 큼 |

## 이 프로젝트에 Flask가 적합한 이유

1. **REST API만 필요**: 모바일/웹앱에서 호출할 API만 필요
2. **빠른 프로토타이핑**: 빠르게 기능을 구현하고 테스트해야 함
3. **단순한 데이터 처리**: 복잡한 데이터베이스 쿼리나 관리자 페이지 불필요
4. **유연한 구조**: 프로젝트 요구사항에 맞게 자유롭게 구조화 가능

## 결론

이 프로젝트는 REST API만 필요하고 빠른 개발이 중요한 프로토타입이므로 Flask가 Django보다 적합합니다. 만약 나중에 더 복잡한 기능(관리자 페이지, 복잡한 데이터베이스 쿼리 등)이 필요하다면 Django로 전환을 고려할 수 있습니다.
