# 기술 선택 이유 (Flask vs Django)

## 🤔 왜 Flask를 선택했나요?

### 1. **프로젝트 요구사항에 적합**
- **REST API만 필요**: 이 프로젝트는 주로 REST API 엔드포인트만 필요합니다
- Django는 풀스택 웹 프레임워크로 템플릿, 관리자 페이지 등 많은 기능을 포함하지만, 이 프로젝트에서는 불필요합니다

### 2. **가벼움과 빠른 개발**
- Flask는 **마이크로 프레임워크**로 최소한의 기능만 제공
- 필요한 것만 추가하는 방식 (필요한 패키지만 설치)
- 빠른 프로토타이핑에 적합

### 3. **단순한 구조**
- 파일 구조가 간단하고 이해하기 쉬움
- 작은 프로젝트에 적합한 구조

### 4. **유연성**
- 원하는 방식으로 구조화 가능
- ORM을 사용하지 않아도 됨 (이 프로젝트는 간단한 데이터 처리)

## 📊 Flask vs Django 비교

| 항목 | Flask | Django |
|------|-------|--------|
| **용도** | 마이크로 프레임워크, API | 풀스택 웹 프레임워크 |
| **학습 곡선** | 낮음 | 높음 |
| **설정** | 최소한 | 많은 설정 필요 |
| **ORM** | 선택사항 (SQLAlchemy) | 내장 (Django ORM) |
| **관리자 페이지** | 없음 (별도 구현) | 내장 |
| **보안** | 수동 설정 | 많은 보안 기능 내장 |
| **확장성** | 작은~중간 프로젝트 | 대규모 프로젝트 |

## 🔄 Django로 변경할 수 있나요?

**네, 가능합니다!** Django로 변경하면:

### Django의 장점
1. **더 나은 구조화**: 앱 단위로 모듈화
2. **내장 보안 기능**: CSRF, XSS 보호 등
3. **ORM**: 데이터베이스 작업이 더 쉬움
4. **관리자 페이지**: 자동 생성되는 관리 인터페이스
5. **확장성**: 대규모 프로젝트에 적합

### Django로 변경 시 필요한 작업
1. Django 프로젝트 구조로 재구성
2. Django REST Framework 추가
3. 모델 정의 (필요시)
4. URL 라우팅 변경
5. 설정 파일 구조 변경

## 💡 현재 프로젝트에 Flask가 적합한 이유

이 프로젝트는:
- ✅ REST API만 제공 (프론트엔드는 React Native)
- ✅ 복잡한 데이터베이스 모델 불필요
- ✅ 빠른 프로토타이핑 필요
- ✅ 가벼운 서버 구조 선호

따라서 **Flask가 더 적합**합니다.

## 🚀 Django로 변경하고 싶으시다면?

Django로 변경을 원하시면:
1. Django 프로젝트 구조로 재구성
2. Django REST Framework 설정
3. 기존 API 엔드포인트를 Django 뷰로 변환
4. 더 나은 보안 및 구조화된 코드

**결론**: 현재 프로젝트는 Flask가 적합하지만, Django로 변경도 가능합니다!
